Console Chat Server

A roguelike monster-collector

Role: Network Systems Programmer

Technologies: C, C++, POSIX Sockets, TCP/IP, UDP, Select, Multithreading, Hash Tables

A console-based chat server capable of handling multiple clients concurrently. Built to demonstrate low-level socket communication, thread-safe design, and secure session management. All interactions follow a custom command protocol, supporting features like user registration, direct and broadcast messaging, and real-time user list retrieval.

Key Contributions

• Implemented multithreaded client handling with select() for efficient in-thread multiplexing..

• Built a custom command interpreter, supporting commands like ~register, ~login, ~send, ~getlist, and ~getlog.

• Designed and developed a secure authentication system using a hashtable for unique user credentials and session management.

• Enabled UDP broadcast functionality for system-wide announcements to all connected clients.

• Structured the application using object-oriented principles, organizing code into reusable components such as ChatServer, ClientHandler, MessageHandler, and User.
